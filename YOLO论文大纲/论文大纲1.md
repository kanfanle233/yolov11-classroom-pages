## 四、创新点（6条，均有代码+实验证据）

### 创新点 1：系统级音视频上下文感知行为分析框架

将YOLOv11行为检测、姿态跟踪、Whisper ASR、教师指令识别、Confidence-Weighted动态融合和LLM语义推理整合为一条可审计的全链路管道。区别于单一检测器改进，本框架强调"检测→跟踪→上下文增强→可解释输出"的系统闭环。

**证据**：`scripts/main/09_run_pipeline.py`（43步管道），6个完整case全部contract=ok。

### 创新点 2：Confidence-Weighted动态音视频融合

提出以实时置信度驱动的动态权重融合公式替代固定权重（0.55/0.45）：
```
p_match = (w_visual × C_visual + w_audio × C_audio) / (w_visual + w_audio)
w_visual = 1 - uq_track    （跟踪不确定性越低，视觉权重越高）
w_audio  = asr_confidence   （ASR质量越高，音频权重越高）
```
无声时w_audio=0，系统自动降级为纯视觉模式。避免了固定权重下视觉兜底事件自匹配（text_score=1.0）的虚假贡献。

**证据**：`verifier/infer.py:134-175`；046.mp4上修复前p_match=0.924（虚高），修复后=0.748（真实纯视觉）。

### 创新点 3：LLM Zero-Shot语义推理填补统计融合边界

首次在课堂行为分析中引入LLM作为零样本语义推理引擎。在37对指令-行为的系统评估中，24%的案例基于关键词规则的统计融合无法识别语义冲突（如"组内交流交流"≠学生应写字），而LLM以平均把握度0.90正确识别了这些冲突。

**证据**：`scripts/pipeline/06f_llm_semantic_fusion.py`；`output/llm_fusion_37_results.json`（Gemini评估）。

### 创新点 4：后排ROI切片推理+跟踪稳定性联合消融

在3个正方视角视频的6变体消融实验中，切片推理使检测学生数+85%（20→37），proxy指标+107%。同时验证了切片/SR不引入身份切换（IDSW=0全部变体）。匈牙利全局匹配进一步将跟踪碎片减少5.4%、大间隙（≥1s）减少27.8%。

**证据**：`output/codex_reports/sr_ablation_paper_summary.json`（16行）；`output/codex_reports/tracking_ablation_comparison.md`。

### 创新点 5：Pipeline Contract可审计链路

设计strict contract机制——每阶段产物必须满足固定schema，否则中断管道。所有case均通过contract验证（unlinked>200k=0）。这使得实验完全可复现、可审计。

**证据**：`codex_reports/.../check_pipeline_contract_v2.py`；6个case全部status=ok。

## 五、方法（简化版）

### 5.1 视觉行为检测与跟踪

- YOLO11s 8类行为检测（tt/dx/dk/zt/xt/js/zl/jz），e150训练，Test mAP50=0.9806
- YOLO11x-pose 17点人体关键点 → ByteTrack风格跟踪 + 匈牙利全局匹配 + 座位先验
- 后排ROI切片推理（rear_adaptive网格）+ opencv SR + artifact_deblur预处理
- 跨切片检测去重（IoU/重叠度/中心距三重阈值）

### 5.2 语音上下文提取

- Whisper small/medium ASR → 分层质量门控 → 简体中文转换
- 音频能量预检（RMS<50跳过ASR，避免静音视频浪费推理）
- 10种教师指令模式匹配（翻开课本/开始写/看黑板/举手/讨论/停笔/介绍/安静/确认听讲/画线）
- 输出：指令类型 + 时间窗口 + 行为增强/衰减规则

### 5.3 音视频融合

**方法一（Confidence-Weighted动态融合）**：
- 视觉分数 = 0.65×overlap + 0.35×action_conf
- 动态权重：w_v=1-uq, w_a=asr_conf
- 无声时w_a=0自动纯视觉；visual_fallback时text_score=NaN

**方法二（LLM语义推理）**：
- 零样本：教师指令文本 + 学生行为标签 → LLM → boost∈[-0.25,+0.25]
- 处理统计融合无法覆盖的隐式语义冲突

### 5.4 Pipeline Contract

- 每步产物schema验证
- strict模式：缺失字段→异常中断
- 6个case全部contract=ok

## 七、讨论与局限

### 诚实局限

1. **语音贡献有限**：大部分正方视角视频<60秒且无声。仅在2/593个视频中提取到有效教师指令。系统设计上支持音视频融合，但实验数据中语音路径的实际贡献受限于数据条件。

2. **GT覆盖不足**：后排GT仅3名学生，导致formal precision极低（1-2%）。论文采用proxy+formal双指标体系透明呈现此限制。

3. **测试集非视频级拆分**：因原始文件名不含视频ID，test split为随机帧级拆分（非按视频）。可能存在同视频不同帧跨split的数据泄漏。

4. **LLM融合为定性评估**：37对系统评估是定性+统计分析，不是机器学习benchmark测试。LLM判断的一致性未经过多模型对比验证。

5. **未与检测器改进方法直接对比**：本工作贡献在系统层面，未与YOLO结构改进方法（如OE-YOLOv8n、VWE-YOLOv8）在相同数据集上直接对比检测精度。
