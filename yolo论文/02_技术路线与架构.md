# 02 - 技术路线与架构

> **最后更新**：2026-05-03

## 整体技术架构

### 核心流水线（Pipeline）

```
视频输入 → YOLO11-Pose(关键点) + YOLO11-Detect(目标)
       → Track & Smooth(轨迹平滑 + 不确定性估计UQ)
       → SlowFast(动作识别) + ASR(语音转写)
       → Object Fusion(目标融合) + Event Query(事件抽取)
       → Multi-modal Align(多模态对齐)
       → Dual Verifier(视觉+语义双重验证)
       → Timeline Visualization(时间线可视化)
```

### 技术栈

| 层次 | 技术 | 用途 |
|------|------|------|
| 姿态估计 | YOLOv11-Pose | 17点人体关键点检测 |
| 目标检测 | YOLOv11-Detect + 自训练Case检测器 | 人/书本/笔/电子设备等检测 |
| 动作识别 | SlowFast R50 | 时序动作分类 |
| 跟踪 | ByteTrack/Bot-SORT | 多人轨迹跟踪 |
| 语音 | Whisper / OpenAI ASR | 课堂语音转文字 |
| 图网络 | IGFormer (DSIG+SDIG) | 多人交互图建模 |
| 多模态 | CCA + DQD | 级联协同注意力 + 动态查询驱动 |
| 验证 | 可学习Verifier + 规则引擎 | 双重验证框架 |
| 前端 | HTML/JS/D3 | 时间线可视化 + 仪表盘 |

### 四大升级方向（来自 implementation_plan.md）

1. **升级一：多人交互图网络** — ST-GCN → IGFormer (DSIG + SDIG)
2. **升级二：同伴感知验证** — 利用周围同学行为上下文修正个体行为
3. **升级三：MLLM + CCA + DQD** — 多模态大语言模型语义验证
4. **升级四：YOLOv11底层增强** — ASPN + DySnakeConv + GLIDE Loss

### 数据流

| 阶段 | 输入 | 输出 |
|------|------|------|
| 01-Pose | 视频 | keypoints.jsonl |
| 02-Objects | 视频 | objects.jsonl |
| 03-Track | keypoints.jsonl | pose_tracks_smooth.jsonl + UQ |
| 05-Action | 视频 + keypoints | actions.jsonl + embeddings.pkl |
| 06-ASR | 音频 | transcript.jsonl |
| 06b-Event | transcript | event_queries.jsonl |
| xx-Align | pose + action + event | aligned_sequences |
| 07-Verify | aligned + visual + text | verified_events.jsonl |
| 10-Timeline | verified_events | 时间线可视化 |
