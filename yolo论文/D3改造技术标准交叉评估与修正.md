### 3.2 数据转换函数规范（必须在前端实现）

由于后端返回的是嵌套对象（`{case_id, events:[...]}`），需要在前端做一次轻量转换：

```javascript
// 在 loadCaseData() 中调用
function normalizeEvents(raw) {
  const events = raw.events || [];
  return events.map(e => ({
    // 视图 A（甘特图）字段
    id: e.event_id,
    track: `S${String(e.track_id).padStart(2,'0')}`,
    start: e.window?.start || e.query_time - 1.5,
    end: e.window?.end || e.query_time + 1.5,
    behavior: e.behavior_code || e.action,
    label: e.label,              // "match" | "uncertain" | "mismatch"
    query_text: e.query_text,    // ASR原文（当前为visual_fallback）
    
    // 视图 B（权重曲线）字段
    wv: e.evidence?.w_visual ?? 1.0,
    wa: e.evidence?.w_audio ?? 0.0,
    fusion_mode: e.evidence?.fusion_mode ?? "unknown",
    
    // 视图 C（雷达图）字段
    evidence: {
      cv: e.evidence?.visual_score ?? 0,
      ct: e.evidence?.text_score ?? 0,
      uq: e.evidence?.uq_score ?? 0.5,
      wv: e.evidence?.w_visual ?? 1.0,
      wa: e.evidence?.w_audio ?? 0.0,
      r: e.reliability_score ?? 0
    },
    
    // 视频联动
    query_time: e.query_time
  }));
}
```

## 4. P0/P1 任务——逐项技术标准评估

### 4.1 P0-2.1 & P0-2.2：点击证据弹窗 + 雷达图

**技术标准的可行性**：✅ 完全可行

**关键修正**：后端 `/api/bundle/{case_id}/verified` **已经返回完整 evidence**，不需要额外开发。前端只需：
1. 甘特图 `<rect>.on("click")` → 取 `d.evidence` 对象
2. 调用 `drawRadar(d.evidence)` → 更新 SVG 雷达
3. `document.getElementById("ev-asr-text").textContent = d.query_text`
4. `d3.selectAll(".behavior-block").classed("dimmed", d => d.id !== clicked.id)`

### 4.2 P0-3.2：Uncertain 脉冲 + 斜纹

**技术标准的可行性**：✅ 方案正确

**关键修正**：`label == 'mismatch'` 也应该用斜纹，不是呼吸灯。正确映射：

| label | 视觉编码 | 理由 |
|-------|---------|------|
| `match` | 正常纯色填充 + 细白边框 | 系统确认 |
| `uncertain` | SVG `<pattern>` 斜纹填充 + 橙色边框 | 置信度不足，需人工判断 |
| `mismatch` | CSS `@keyframes` 红色脉冲呼吸 + 粗红边框 | 明确冲突，最高告警 |

```css
/* mismatch 脉冲 */
@keyframes pulse-mismatch {
  0%, 100% { filter: drop-shadow(0 0 2px rgba(231,76,60,0.4)); }
  50%      { filter: drop-shadow(0 0 12px rgba(231,76,60,0.9)); }
}
.behavior-block.mismatch {
  animation: pulse-mismatch 0.8s ease-in-out infinite;
}
```

### 4.3 P1-3.1：模态权重博弈曲线

**技术标准的可行性**：✅ 方案正确

**关键修正**：后端 verified_events 中很多事件的 `w_audio = 0.0`（无声视频的纯视觉兜底）。**权重曲线在无音频case上是一条平线**——必须在有音频的 case（45618）上才有交叉博弈效果。

### 4.4 Case 切换逻辑

**技术标准的可行性**：✅ 方案正确

**修正**：使用已有 API `/api/bundle/list` 获取可选 case 列表，填充 `<select>`：

```javascript
async function loadCaseList() {
  const list = await fetch('/api/bundle/list').then(r => r.json());
  const select = document.getElementById('case-selector');
  select.innerHTML = list.map(c => 
    `<option value="${c.case_id}">${c.case_id} (${c.students} students, ${c.contract_status})</option>`
  ).join('');
  select.addEventListener('change', e => loadCaseData(e.target.value));
}
```
