## 二、需要纠正/适配之处 ⚠️

### 问题 1：数据结构不匹配（最关键）

**第三方方案的mock数据格式**：
```json
{ time: 0, duration: 3, v_label: 'tt', a_trigger: '...', wv: 0.85, wa: 0.4, uncertain: false }
```

**项目真实数据格式**（来自 `verified_events.json` 和 `timeline_students.json`）：

```json
// timeline_students.json 中的 segment
{
  "student_id": "S01",
  "track_id": 1,
  "start_time": 0.0,
  "end_time": 3.2,
  "behavior_code": "tt",
  "behavior_label_zh": "抬头听课",
  "confidence": 0.87
}

// verified_events.json 中的 event
{
  "event_id": "v_000000",
  "track_id": 12,
  "query_text": "visual_fallback:listen/听课",
  "query_time": 14.42,
  "window": {"start": 13.355, "end": 15.485},
  "p_match": 0.989,
  "reliability_score": 0.983,
  "label": "match",
  "evidence": {
    "visual_score": 0.981,
    "text_score": 1.0,
    "uq_score": 0.011,
    "w_visual": 1.0,
    "w_audio": 0.0
  }
}
```

**后果**：第三方代码中 `d.wv`、`d.wa`、`d.uncertain`、`d.a_trigger` 都需要重新映射到真实字段名。

### 问题 2：真实数据中 uncertain 零样本

**第三方方案**假设有 `uncertain: true` 的案例来展示红色斜纹。但 paper mainline 的12条 verified_events **全部是 `match`**（因为用的是无音频视频，纯视觉兜底模式）。

**后果**：斜纹阴影功能写好了也无法展示真实效果，需要：
- 方案A：用 45618（有音频的case）重跑 Pipeline 产生真实的 uncertain
- 方案B：在demo模式下注入模拟冲突数据

### 问题 3：真实 ASR 文本是 visual_fallback

**第三方方案** `a_trigger: '大家把笔记记一下'` 是中文教学指令。但 paper mainline 的 `query_text` 全是 `"visual_fallback:listen/听课"`——**没有真实ASR文本**。

**后果**：音频标记的tooltip打不出真实指令，需要：
- 用 45618 的 ASR 数据重跑
- 45618 的 `instruction_context.json` 已有12条真实指令（"我们要是换直线的时候就用直纸"等）

### 问题 4：模拟数据只有5条

**项目真实数据量**：
- 046_A8：276条timeline行，12条verified events，37个学生
- 45618_sliced：2179条timeline行，32条verified events，45个学生

第三方方案的5条硬编码数据仅适合做概念验证，实际页面需要动态加载 JSON。

## 四、集成方案：第三方代码 → 项目实际代码的改造路径

### 需要保留的项目已有实现

```
paper_demo.html 已有且成熟的部分：
├── 甘特图 (drawTimeline)           ← 改造：增加uncertain斜纹 + click证据弹窗
├── 消融柱状图 (drawAblation)       ← 保留不变
├── 指标面板 (drawMetrics)          ← 保留不变
├── 视频播放 + click-to-seek        ← 保留 + 增强
├── 颜色映射 (BEHAVIOR_COLORS)      ← 保留（8色体系比mock的3色更完整）
└── 8类中文标签 (BEHAVIOR_LABELS)   ← 保留
```

### 需要从第三方方案新增的部分

```
新增功能（需对接真实数据字段）：
├── 权重折线图                 ← 新模块，数据源：verified_events[].evidence
├── Uncertain斜纹阴影          ← 甘特图增强，数据源：verified_events[].label
├── 音频标记点                 ← 新增，数据源：event_queries.jsonl
├── 证据侧边弹窗               ← 新面板，数据源：verified_events[] + behavior_segments
├── D3雷达图                   ← 新面板子组件
└── d3.brushX() 刷取缩放      ← 折线图交互
```

### 字段映射表（Mock → 真实）

| Mock字段 | 真实数据源 | 真实字段路径 |
|----------|-----------|-------------|
| `d.time` | timeline_students.json | `segments[].start_time` |
| `d.duration` | timeline_students.json | `segments[].end_time - start_time` |
| `d.v_label` | timeline_students.json | `segments[].behavior_code` |
| `d.a_trigger` | verified_events.json | `events[].query_text`（当前为visual_fallback） |
| `d.wv` | verified_events.json | `events[].evidence.w_visual` |
| `d.wa` | verified_events.json | `events[].evidence.w_audio` |
| `d.uncertain` | verified_events.json | `events[].label === 'uncertain'` |

## 五、可行性总评

| 维度 | 评分 | 说明 |
|------|------|------|
| **技术方案正确性** | ⭐⭐⭐⭐⭐ | D3 API使用、数据结构设计、交互逻辑均正确 |
| **与项目真实数据对接** | ⭐⭐⭐ | 需要做字段映射；关键数据（uncertain、真实ASR）当前缺失 |
| **代码可直接复用度** | ⭐⭐ | 骨架可参考，但所有数据绑定需重写以对接真实JSON |
| **录屏展示建议** | ⭐⭐⭐⭐⭐ | 分镜策略完全正确——"鼠标划过→证据弹出→uncertain暂停说明" |

**综合结论**：方向完全正确，但**不能直接复制粘贴用**。需要以项目现有 `paper_demo.html` 为基础，将第三方方案的：
- 折线图逻辑 → 对接 `verified_events[].evidence`
- 斜纹模式 → 对接 `verified_events[].label`
- 音频标记 → 对接 `event_queries[].trigger_text`
- 证据弹窗 → 融入甘特图click事件

改造量约 **2-3天**，核心瓶颈仍是**真实数据中缺乏uncertain/真实ASR样本**，需要先用45618有音频case重跑Pipeline。
