# 智慧课堂学生行为数据集概览与数据清洗指南

此文档旨在为后续的 AI 模型训练（如 YOLOv11 行为检测微调）提供关于 `data/智慧课堂学生行为数据集` 的结构解析、内容说明及数据清洗与预处理的重点注意事项。

## 1. 数据集背景与整体结构

该数据集的原始数据来源于国家资源公共服务平台中2019年度小学语文1至6年级的部级优课视频，共计 131 节。为了便于深度学习处理，原始长视频已由 FFmpeg 根据镜头切换自动均匀切割为无转场短片段。

**目录结构与视角落点分布**：
```text
data/智慧课堂学生行为数据集/
├─ readme.txt                 # 数据集全局说明
├─ 正方视角/                  # 正面拍摄视频 (593个片段，时长分布 1-174s)
├─ 后方视角/                  # 教室后方拍摄视频 (4544个片段，时长分布 1-59s)
├─ 斜上方视角1/               # 教室斜上方角度拍摄视频之一
├─ 斜上方视角2/               # 教室斜上方角度拍摄视频之二 (与视角1合计 4781个片段，时长 1-115s)
├─ 教师视角/                  # 讲台看向学生的教师视角 (2918个片段，时长 1-106s)
└─ 案例/                      # 抽取帧与标准 JSON 标注目录
   ├─ README.txt              # 标注类别说明与数量统计
   └─ 智慧课堂学生行为数据集案例（正方视角）/ # 包含按 1fps 提取的 .jpg 结构及同名 .json 标签
```

## 2. 标注数据内容解析（基于“案例”目录）

目前核心的 **已标注训练数据** 集中在 `案例/智慧课堂学生行为数据集案例（正方视角）/` 文件夹内，它包含了按 `1 帧/秒` 密度提取的 `.jpg` 图像以及对应的 `.json` 标签文件。

### 2.1 行为类别与数据分布

共包含 8 种学生课堂行为，整体呈现典型的**极度长尾分布**（类别不平衡）：
- **基线行为（主流占比较大）**：
  - `tt`：抬头听课 (117,528)
  - `dx`：低头写字 (72,462)
  - `dk`：低头看书 (58,932)
- **偶发/动态交互行为（占比极小）**：
  - `zt`：转头 (5,339)
  - `xt`：小组讨论 (4,663)
  - `js`：举手 (4,183)
  - `zl`：站立 (4,101)
  - `jz`：教师指导 (680)

### 2.2 JSON 标签格式

单张图片的标注格式为自定义 JSON 字典，内部列表通过 `labels` 键存储：
```json
{"labels": [
    {"name": "tt", "x1": 232, "y1": 121, "x2": 260, "y2": 158},
    {"name": "js", "x1": 11, "y1": 231, "x2": 81, "y2": 308}
]}
```
**字段说明**：
- `name`：以字符缩写代表的类别（tt, dx, zl 等）。
- `x1, y1`：标注框左上角绝对像素坐标。
- `x2, y2`：标注框右下角绝对像素坐标。

## 3. 下一步：数据清洗与预处理（重点注意事项）

为使这些数据顺利被 YOLOv11 或本项目中的 `intelligence_class/training/` 管道利用，在启动数据加载前，必须关注以下清洗与转换步骤：

### 3.1 格式转换 (Format Conversion)
YOLO 模型不接受该坐标表示方法。必须编写转换脚将 `[x1, y1, x2, y2]` 转换为 YOLO 所需的 `[class_id, x_center, y_center, width, height]`（即中心坐标宽高并相对于图片长宽进行 0~1 的归一化）。同时需要将字符型 `name` 映射为连续的数字 ID（0-7）。

### 3.2 类别极其不平衡的处理 (Data Imbalance)
如数据分布所示，“抬头听课 `tt`”有接近 12 万条，而“教师指导 `jz`”仅不到 700 条。
**应对策略**：
1. **Focal Loss / 类权重池**：在训练参数中强制赋予少数类更高的 Loss 惩罚。
2. **数据增强 (Data Augmentation)**：对包含 `jz`, `zl`, `js` 等少数类的图片实施定向裁剪、颜色抖动、水平翻转（`training/02_dataset_augment...` 的用武之地）。

### 3.3 严格的数据集划分控制 (防止 Data Leakage)
因为这些图像是从同一批视频中按 1 fps 密集抽取的，**相邻帧之间的人体特征高度冗余与相似**。
**极其关键**：
在划分训练集 (Train) 和验证集 (Val) 时，**绝对禁止**纯粹打乱图片后随机划分。必须**按照视频级别划分**（例如前 80% 的源视频全部划入训练集，剩余 20% 的源视频划入验证集）。否则模型会“见过”验证集中的同一批学生，导致训练和测试指标虚高，在全新课堂中泛化崩溃。

### 3.4 视角泛化鲁棒性评估
目前提取的标注主要来源于“正方视角”，但原始视频集有大量“斜上方”、“后方”等高畸变视角。若模型直接部署在多视角场景下，可能会产生检测衰减。
若未来计划纳入另外 3 类视角的未标注视频，可以采用如一致性学习、伪标签自训练（Pseudo-labelling）等半监督/弱监督手段提升在全局视角的工程可用性。
