# 课堂行为多模态分析系统——程序介绍报告

## 1. 项目定位与目标
本项目是一个面向课堂场景的**多模态行为分析系统**，核心目标是：
- 从课堂视频中自动提取学生与群体行为；
- 结合语音转写（ASR）进行时序对齐与语义校验；
- 输出可视化结果，支持教师复盘、教学评估与行为洞察。

系统名称和主入口在 README 中已明确，采用“YOLOv11 + 时序逻辑 + 可视化”的流水线式设计，便于分步调试与扩展。 

---

## 2. 系统总体架构

### 2.1 核心思想
系统采用**“离线分析流水线 + 在线可视化服务”**双层结构：
1. `scripts/09_run_pipeline.py` 负责任务编排，串联多个步骤并支持缓存复用；
2. `server/app.py` 提供 FastAPI 服务，统一读取输出目录中的结果文件并驱动前端展示；
3. `web_viz/templates/index.html` 提供多面板可视化界面（时间轴、视频、统计、转写、投影等）。

### 2.2 路径与目录约定
`paths.py` 统一定义了项目根目录、数据目录、输出目录、模型目录等路径，保证脚本和服务读取同一套目录规范，减少路径硬编码带来的维护成本。

---

## 3. 数据处理流水线（Pipeline）说明
主调度脚本：`scripts/09_run_pipeline.py`。

### 3.1 编排机制
- 使用 `run_step()` 以子进程调用各步骤脚本；
- 使用 `maybe_run()` + `file_is_fresh()` 判断输出文件是否“新鲜”，实现**缓存跳过**；
- 支持 `--force` 强制重跑、`--from_step` 断点续跑、`--skip_video/--skip_vis` 控制耗时步骤。

这意味着该系统不仅是“能跑通”，还考虑了工程效率与可重复实验。

### 3.2 关键步骤概览
按脚本内定义，主要过程包括：
1. **姿态关键点提取（Step 02）**：导出人体关键点序列；
2. **检测与演示视频（Step 21 / Step 03 / Step 09）**：生成人体/目标检测结果与演示视频；
3. **轨迹平滑（Step 04）**：对人体轨迹进行跟踪与平滑，降低抖动；
4. **动作识别（Step 05）**：SlowFast 行为识别，并可输出 embedding 与关键帧；
5. **语音转写（Step 06）**：将课堂音频转成带时间戳文本；
6. **双重校验融合（Step 07）**：融合动作与语音，生成每人行为序列，支持同伴感知参数；
7. **可选 MLLM 语义复核（Step 14）**：对行为片段做更高层语义验证；
8. **群体交互建模（Step 11）**：输出群体事件；
9. **特征与投影（Step 12/13）**：生成学生级特征与二维投影结果；
10. **叠加视频与时间轴（Step 08/10）**：产出可播放叠加视频和时间轴图。

### 3.3 输出组织
每个视频在 `output/<name>/` 下形成完整结果包（jsonl/json/png/mp4 等），可直接被可视化服务消费，形成“分析-展示”闭环。

---

## 4. 在线可视化与服务层
服务主文件：`server/app.py`。

### 4.1 服务职责
- 启动 FastAPI 应用并配置 CORS；
- 挂载 `/output`、`/data`、`/videos`、`/static` 等静态路径；
- 缓存读取 timeline、stats、transcript、tracks 等结构化结果；
- 计算行为投影（支持 `pca` / `mds` / `tsne`，以及 `euclidean` / `levenshtein` 距离）。

### 4.2 工程细节亮点
- 使用 `lru_cache` 减少重复 IO 和重复计算；
- 内置 `Levenshtein` 缺失时的兜底实现，提升环境兼容性；
- 对 `paths.py` 采用“优先导入+回退推断”策略，提高部署鲁棒性。

---

## 5. 前端交互界面说明
前端模板：`web_viz/templates/index.html`。

### 5.1 布局结构
页面采用 Grid 多区域布局，包含：
- 顶部状态栏；
- 左侧视频/对象列表；
- 中央主视图（多 Tab）；
- 右侧统计洞察面板；
- 底部视频播放区 + 时间轴可视化区。

### 5.2 展示能力
- 使用 D3 渲染时序和统计图形；
- 与视频播放联动（含覆盖层 canvas）；
- 支持转写文本、投影点、图例等多信息同屏呈现。

该界面更偏“教学分析工作台”形态，适合老师进行回看、比较与定位关键行为片段。

## 6. 我向老师汇报时可强调的创新点
1. **多模态融合**：视频动作、对象、语音文本、群体事件统一到时间轴；
2. **分层验证**：规则/模型基础上可接入 MLLM 做语义复核，降低误判；
3. **工程可复用**：支持缓存跳步、断点续跑，适合多视频批处理；
4. **可解释可回放**：不仅输出分类结果，还可回放视频、查看证据链；
5. **可扩展接口**：从脚本化流程到 FastAPI 服务，便于后续接 Web 平台。

## 8. 汇报总结（可直接朗读）
本项目已经具备“**算法处理链路 + 可视化解释平台**”的完整雏形。它不是单点识别模型，而是一套面向课堂场景的多模态分析系统：前端可看、后端可跑、结果可追溯。对教师而言，价值在于更高效地复盘课堂与定位关键行为；对研究与工程而言，价值在于可迭代、可扩展、可部署。下一步若补齐评测与部署规范，这套系统具备进一步落地应用的潜力。
