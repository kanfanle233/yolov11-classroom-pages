## 结论先行

- `front_046` 已经进入 formal GT 评估口径，`gt_status=ok`，可以同时报告 proxy 与 formal 指标。
- `front_001/front_002` 仍是 `gt_status=missing`，这些 case 只能支撑后排覆盖、行为挂载、运行成本等 proxy 结论。
- 后排增强不能只看“检出更多学生”。`A8` 覆盖更多，但 `A3` 才是当前 `front_046` 上 formal recall 最优的变体。
- 论文中应把“检测能力提升”和“正式准确率提升”拆开写，不能把 proxy 百分比直接当 accuracy。

## 当前论文写作边界

### 可以直接写

- 行为检测 test 指标。
- `front_046` 上 formal recall / IDF1 / HOTA / behavior_macro_f1。
- `A8` 带来更高覆盖，`A3` 带来更高 formal recall，这两者并不相同。
- 外部 SR 后端缺失时，实验条目应标记为 unavailable，而不是已完成。

### 不能直接写

- 不能把 `front_001/front_002` 的 proxy 指标当作正式准确率。
- 不能把 `A8` 直接宣称为“总体最优”，因为 formal 指标并不支持这个结论。
- 不能把 `PCK/OKS` 写成当前主结论，因为现有 `front_046` GT 未标关键点。
- 不能把 `front_046` 的低 precision 当成“检测器失效”。当前 GT 只覆盖 3 名学生，precision 主要反映 GT 覆盖有限。

## 当前最稳妥的论文表述

- “后排切片与增强显著提高候选覆盖，formal GT 在 `front_046` 上验证了对已标注学生的 recall 改善。”
- “A3 在 formal recall 上最佳，A8 在候选覆盖上最佳，说明覆盖率与身份稳定性之间仍存在权衡。”
- “front_001/front_002 当前主要作为 proxy 与工程口径的补充 case，不承担正式准确率结论。”
