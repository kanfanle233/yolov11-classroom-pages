## GT 文件的最小字段

`persons[]` 最少需要：

- `bbox`
- `track_id`

若要扩展更多指标，可继续补：

- `behavior_code`
- `actions`
- `keypoints`

当前影响关系：

| 字段 | 支撑指标 |
| --- | --- |
| `bbox + track_id` | person recall/precision/F1、AP、IDF1、IDSW、HOTA、MOTA |
| `behavior_code` | behavior F1 |
| `actions` | temporal mAP |
| `keypoints` | PCK / OKS |

## 使用注意

- 不要拿 `046.mp4` 的 GT 去评估 `002.mp4` 的 case。
- 不要把 `front_002_rear_gt.jsonl` 直接当成当前 formal 主样本。
- 当前 `front_046` GT 可以支撑 formal person/tracking/behavior 结论，但不能支撑 keypoint 结论。
- 若后续扩展 GT，应优先补更多学生覆盖，而不是先补更多视频数量。
