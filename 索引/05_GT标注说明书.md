# 05 后排 GT 标注说明书

## 为什么必须人工标注 GT

GT 是 `Ground Truth`，也就是人工确认的真实答案。模型输出不能自己当答案，否则会出现“拿预测评估预测”的泄漏，指标会虚高，最典型就是很多值变成 `1.0`。

GT 的用途：

- 区分 `TP/FP/FN`：没有人工框，就不知道漏检了谁、误检了谁。
- 计算检测准确率：`precision/recall/F1/AP50/AP75/mAP50-95` 都需要真实人框。
- 计算姿态准确率：`PCK/OKS` 需要真实关键点，否则只能看“关键点数量”这种 proxy。
- 计算身份稳定性：`IDF1/IDSW/HOTA/MOTA` 需要同一学生跨帧的真实 `track_id`。
- 计算行为准确率：`behavior_macro_f1/frame_mAP/temporal_mAP` 需要真实行为类别和动作起止时间。
- 支撑论文结论：没有 GT，只能说“召回代理指标提升”，不能严谨说“准确率提升”。

## 最小标注范围

建议先标注 `front_002` 后排 ROI 的 30-60 秒。

| 目标 | 最小标注 | 可计算指标 |
| --- | --- | --- |
| 检测 | 每秒 1 帧的人框 bbox | precision/recall/F1、AP50/AP75/mAP |
| 姿态 | 每秒 1 帧的人框 + 17 个 COCO 关键点 | PCK、OKS |
| 跟踪 | 连续帧或每 5 帧一个稳定 track_id | IDSW、IDF1、HOTA/MOTA 近似 |
| 行为帧分类 | 每秒 1 帧 behavior_code | behavior macro/micro F1、frame-mAP |
| 行为时间段 | start/end + behavior_code + track_id | segment IoU、temporal mAP@0.3/0.5/0.7 |

如果只标 1 FPS，跟踪指标只能当 sparse tracking 评估；如果要严格 IDF1/HOTA，应尽量标连续帧或更高频率采样。

## JSONL 标注格式

每一行是一帧：

```json
{
  "frame": 250,
  "t": 10.0,
  "roi": [0, 0, 1280, 446],
  "image": "output/codex_reports/rear_gt/gt_frames/frame_000250.jpg",
  "persons": [
    {
      "bbox": [420, 105, 455, 178],
      "track_id": 7,
      "keypoints": [[430, 112, 1], [428, 110, 1], [432, 110, 1]],
      "behavior_code": "dk",
      "occlusion": "partial"
    }
  ],
  "actions": [
    {
      "track_id": 7,
      "behavior_code": "dk",
      "start_frame": 250,
      "end_frame": 375,
      "start_time": 10.0,
      "end_time": 15.0
    }
  ]
}
```

## 字段规则

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| `frame` | 是 | 原视频帧号。 |
| `t` | 建议 | 秒级时间，便于核查视频。 |
| `roi` | 建议 | 后排 ROI，评估器会再次按 ROI 过滤。 |
| `persons[].bbox` | 是 | `[x1,y1,x2,y2]`，原视频像素坐标。 |
| `persons[].track_id` | 是 | 视频内稳定学生 ID，同一个学生跨帧保持不变。 |
| `persons[].keypoints` | 可选 | COCO 17 点，格式 `[[x,y,v], ...]`。不标则 PCK/OKS 为空。 |
| `persons[].behavior_code` | 可选 | 8 类行为代码。不标则 behavior F1 为空。 |
| `persons[].occlusion` | 可选 | `none/partial/heavy`，用于后续按遮挡程度分组分析。 |
| `actions[]` | 可选 | 行为时间段 GT，用于 segment IoU 和 temporal mAP。 |

## COCO 17 关键点顺序

| index | keypoint |
| --- | --- |
| 0 | nose |
| 1 | left_eye |
| 2 | right_eye |
| 3 | left_ear |
| 4 | right_ear |
| 5 | left_shoulder |
| 6 | right_shoulder |
| 7 | left_elbow |
| 8 | right_elbow |
| 9 | left_wrist |
| 10 | right_wrist |
| 11 | left_hip |
| 12 | right_hip |
| 13 | left_knee |
| 14 | right_knee |
| 15 | left_ankle |
| 16 | right_ankle |

`v` 建议：`0=不可见/不标`，`1=遮挡但可推断`，`2=可见`。当前评估器把 `v>0` 作为有效关键点。

| code | semantic_id | 英文含义 |
| --- | --- | --- |
| `tt` | `listen` | head-up listening |
| `dx` | `write` | writing |
| `dk` | `read` | reading |
| `zt` | `turn_head` | turning head |
| `xt` | `group_discussion` | group discussion |
| `js` | `raise_hand` | raise hand |
| `zl` | `stand` | standing |
| `jz` | `teacher_interaction` | teacher interaction |

## 标注质量规则

- 不要用模型输出直接复制成 GT；可以用模型输出辅助，但必须人工核查和修正。
- 后排被遮挡但能确定是同一学生时，保留同一个 `track_id`。
- 看不清行为时不要硬标行为，留空比错标更好。
- 行为类别冲突时以主行为为准，例如低头看书和写字难区分时，优先看手部是否在书写。
- 多人重叠时 bbox 框住可见人体区域；不要把两个人框成一个人。
- 同一帧内 `track_id` 不得重复。
- 如果学生短暂消失后回到同一座位且外观/位置连续，应继续使用原 `track_id`。

## 验收标准

- `gt_status=ok`。
- `gt_persons > 0`。
- `person_recall/person_precision/person_f1` 不为空。
- 如果标了 keypoints，`PCK_0_10/OKS_AP` 不为空。
- 如果标了连续 `track_id`，`IDF1/IDSW/HOTA/MOTA` 不为空。
- 如果标了 `behavior_code`，`behavior_macro_f1` 不为空。
- 如果标了 `actions[]`，`segment_iou/temporal_mAP_0_5` 不为空。

## 常见错误

| 错误 | 后果 |
| --- | --- |
| bbox 使用裁剪坐标 | IoU 全错，检测指标异常低。 |
| 用预测结果直接当 GT | 指标虚高，很多值变成 1。 |
| 同一学生跨帧换 track_id | 人为制造 IDSW，IDF1/HOTA 下降。 |
| 只标清楚学生，不标漏检学生 | recall 虚高。 |
| 不标误检区域附近的真实空白 | precision 无法可信。 |
| 不标 actions 起止时间 | temporal mAP 无法计算。 |
