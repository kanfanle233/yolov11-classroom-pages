# 07 后端模型与算法验收标准

## 验收原则

“完美完成”不能只看视频主观观感，也不能只看单个 mAP。需要同时满足：

- 人眼观感：可读、少遮挡、ID 稳定、行为标签不乱跳。
- 正式 GT 指标：检测、姿态、跟踪、行为、后排增强都有量化结果。
- 流程合同：每一步产物能被下一步读取，不能靠手工改文件。
- 运行成本：耗时、显存、缓存大小在可接受范围内。

## 三档完成标准

| 级别 | 定义 | 适用场景 |
| --- | --- | --- |
| Bronze | 工程可跑通 | 内部 demo，流程验证 |
| Silver | 论文可用 | 有 GT、有消融、有对比、有失败分析 |
| Gold | 展示/答辩稳定 | 指标达标，前端可解释，跨视频稳定 |

## 人眼观感标准

| 项目 | Bronze | Silver | Gold |
| --- | --- | --- | --- |
| 骨架显示 | 前排基本正确 | 前中排稳定，后排有明显改善 | 后排主要学生也能稳定显示 |
| 学生 ID | 大部分不跳 | 抽查 5-10 个学生少跳 | 30-60 秒内核心学生无明显跳变 |
| 行为标签 | 能显示 8 类行为 | 标签不大面积闪烁，不遮挡主体 | 时间段标签稳定，能解释每个学生行为 |
| 融合视频 | 能播放 | 标签裁剪正确，未匹配框灰色 debug | 视频可直接用于论文/答辩展示 |
| 后排增强 | contact sheet 可看出差异 | 后排候选更多且错检不明显增加 | 指标与视觉同时证明收益 |

## 正式指标建议阈值

这些阈值是当前项目的阶段目标，不是通用论文标准。真实阈值应以后排 GT 难度调整。

| 指标 | Bronze | Silver | Gold | 说明 |
| --- | --- | --- | --- | --- |
| `rear_person_recall` | >= 0.70 | >= 0.80 | >= 0.88 | 后排人框召回，优先保证不漏人 |
| `person_precision` | >= 0.75 | >= 0.82 | >= 0.88 | 防止切片/SR 产生大量误检 |
| `AP50` | >= 0.70 | >= 0.80 | >= 0.88 | 人框 IoU=0.5 下 AP |
| `mAP50_95` | >= 0.40 | >= 0.50 | >= 0.60 | 后排小目标较难，不宜设太高 |
| `PCK_0_10` | >= 0.60 | >= 0.72 | >= 0.80 | 后排关键点准确率 |
| `OKS_AP` | >= 0.45 | >= 0.55 | >= 0.65 | 需用标准工具复核 |
| `IDF1` | >= 0.75 | >= 0.85 | >= 0.90 | 身份稳定核心指标 |
| `IDSW` | 比 A0 低 | <= A0 | 接近 0 | 允许少量遮挡切换，但不能比 baseline 差 |
| `HOTA` | >= 0.55 | >= 0.65 | >= 0.75 | 检测与关联综合指标 |
| `track_gaps_ge_1s` | ≤ 40 | ≤ 20 | ≤ 10 | ≥1 秒跟踪间隙数（TK-1: 26） |
| `track_fragments` | ≤ 5 | ≤ 2 | 0 | 同一学生被分配多个 track_id 的案例数 |
| `cx_stdev_mean` | ≤ 10 | ≤ 7 | ≤ 5 | 横向位置稳定性（px），越小越好 |

### 跟踪消融验证标准（新增）

| 对比 | 必须观察 | 验收标准 |
|---|---|---|
| TK-0 vs TK-1（匈牙利匹配）| 碎片 track 减少 | tracks 数下降，≥1s 间隙 -20%+ |
| TK-1 vs TK-2（Kalman 滤波）| 间隙减少 | 如无改善则诚实报告负结果 |
| 自定义 vs ByteTrack | 平滑度 vs 检测量 | ByteTrack 间隙更少但检测量低 30-50% |
| `behavior_macro_f1` | >= 0.60 | >= 0.72 | >= 0.80 | 8 类行为类别均衡指标 |
| `temporal_mAP_0_5` | >= 0.45 | >= 0.60 | >= 0.70 | 行为时间段定位 |
| `unlinked_rows_ge_200000` | 0 | 0 | 0 | hybrid 主线硬约束 |
| `student_track_students == tracked_students` | 必须 | 必须 | 必须 | 身份主干一致 |

## 后排增强收益标准

后排增强只有在“正式指标提升”时才能作为论文主创新点。只看图片变清楚不够。

| 对比 | 必须观察 | 合格标准 |
| --- | --- | --- |
| A0 full vs A1 sliced | 切片是否提升后排召回 | `rear_person_recall` 或 `rear_pose_person_rows_proxy` 提升，precision 不明显下降 |
| A1 sliced vs A2 opencv SR | 简单放大是否有效 | recall/PCK 提升或持平，runtime 增加可接受 |
| A2 vs A3 artifact_deblur | 去噪/去模糊是否有效 | 小目标稳定性提升，错检不明显增加 |
| A3 vs A8 adaptive | 自适应 ROI 是否有效 | 后排召回提升，同时 IDF1/HOTA 不下降 |
| A8 vs A4/A5/A6 | 深度 SR/VSR 是否值得 | 正式指标提升大于工程成本，否则只作为负结果消融 |

## 模型与算法还需要做到什么程度

### Pose 主线

- `01_pose_video_demo.py` 与 `02_export_keypoints_jsonl.py` 的 `conf/imgsz/device/half` 参数一致。
- `pose_tracks_smooth.jsonl` 能覆盖课堂主要学生。
- 后排 ROI 内不要因为规则太严过早删除候选。
- 平滑应在检测/跟踪后做，不能在导出前把低置信人全部删掉。

### 8 类行为检测

- 当前模型先作为行为框 detector，不建议改 YOLO head。
- 如果 `behavior_macro_f1` 低，先补样本和微调，而不是改流程接口。
- 重点补 `jz/js/zl/xt`、后排小目标、遮挡、低头读写混淆。

### Hybrid 绑定

- pose track 是唯一学生身份主干。
- 行为框只作为 `behavior_code/semantic_id/conf` 属性。
- 未匹配行为框写入 `behavior_unmatched.jsonl`，只用于 debug 和前端灰色虚线显示，不进入 timeline/fusion。

### 跟踪与座位先验

- 如果 IDSW 是主要错误，优先调 tracking/seat prior/ReID。
- 如果漏检是主要错误，优先调 sliced/SR/模型微调。
- 如果行为错分是主要错误，优先调行为模型和 temporal smoothing。

## 不建议作为完成依据的情况

- 只看一张截图觉得效果好。
- 只看 smoke test 的 `1.0`。
- 只看 `rear_pose_person_rows_proxy` 增加，不看 precision/IDSW。
- 只看 SR 后图片更清晰，不看下游检测/跟踪/行为指标。
- 只跑一个 front 视频，不做跨视频复核。
