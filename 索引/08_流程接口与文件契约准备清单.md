## Schema 检查清单

| 文件 | 检查规则 |
| --- | --- |
| `pose_keypoints_v2.jsonl` | 每行 `frame` 唯一；`persons` 可为空但字段存在；bbox 为原图坐标 |
| `pose_tracks_smooth.jsonl` | `track_id` 为 int；同帧同 track 不重复；keypoints 不丢字段 |
| `behavior_det.semantic.jsonl` | `behavior_code` 必须属于 8 类之一；`semantic_id` 非空 |
| `student_tracks.jsonl` | hybrid 下 `track_id` 必须来自 pose；不允许生成 `200000+` 主线 ID |
| `actions.behavior.semantic.jsonl` | 同一 `track_id` 时间段按时间排序；`end_time >= start_time` |
| `actions.fusion_v2.jsonl` | hybrid 下不允许 `track_id>=200000` |
| `timeline_students.csv` | student_id、track_id、start/end、behavior_code、confidence、source 非空 |
| `rear_row_metrics.json` | 有 GT 时 `gt_status=ok`；无 GT 时必须明确 `missing` |

## 接口断链预警

| 问题 | 典型表现 | 处理 |
| --- | --- | --- |
| 01 效果好但 02 差 | `pose_keypoints_v2.jsonl` 人数明显少 | 检查 `pose_conf/imgsz` 是否与 01 一致 |
| 行为框抢错人 | student track 行为跳变 | 行为 bbox 扩展匹配、Hungarian assignment、segment smoothing |
| 未链接行为污染 timeline | `track_id>=200000` 出现 | hybrid 下禁止未链接行为进入 fusion |
| 后排增强提高召回但 IDSW 上升 | A8 track gap 增加 | 加 seat prior/ReID 或降低 dense 策略强度 |
| 前端图表缺数据 | JSON/CSV 字段名不统一 | 先生成 `frontend_data_manifest.json` 和 schema 文档 |
