# 09 前端 D3 可视化实施方案

## 前端目标

前端不是只播放视频，而是把“模型做了什么、为什么可信、哪里失败、后排增强是否有效”解释清楚。

核心页面建议：

- Demo 总览页
- 单视频分析页
- 学生时间线页
- 后排增强消融页
- 指标评估页
- 失败案例页
- 数据与接口文档页

## 页面结构建议

### 1. Demo 总览页

展示内容：

- 当前 case ID、视频、模型、参数。
- contract 状态：`pipeline_status`、`fusion_status`。
- 核心 KPI：`tracked_students`、`student_track_students`、`unlinked_rows_ge_200000`、`matched_behavior_items`。
- 快捷入口：打开融合视频、打开时间线、打开消融结果。

数据源：

- `pipeline_manifest.json`
- `pipeline_contract_v2_report.json`
- `fusion_contract_report.json`
- `student_tracks.report.json`
- `pose_behavior_video.report.json`

### 2. 单视频分析页

展示内容：

- 左侧 HTML video 播放 `pose_behavior_fusion_yolo11x.mp4`。
- 右侧 D3 时间线显示学生行为段。
- 点击时间线某段，视频跳到 `start_time`。
- 点击学生 ID，高亮该学生轨迹和行为段。

数据源：

- `pose_behavior_fusion_yolo11x.mp4`
- `timeline_students.csv`
- `student_tracks.jsonl`
- `actions.behavior.semantic.jsonl`

### 3. 后排增强消融页

展示内容：

- A0/A1/A2/A3/A8 的指标柱状图。
- 后排 contact sheet 图片。
- runtime/cache 成本图。
- 一键切换 proxy 指标和 formal GT 指标。

数据源：

- `output/codex_reports/sr_ablation_paper_summary.json`
- `output/codex_reports/front_001_sr_ablation/sr_ablation_contact_sheet.jpg`
- `output/codex_reports/front_002_sr_ablation/sr_ablation_contact_sheet.jpg`
- 各 case `rear_row_metrics.json`

### 4. 指标评估页

展示内容：

- 检测、姿态、跟踪、行为、SR、工程性能 6 类指标。
- smoke/proxy/formal 三种指标状态明确区分。
- 若 `gt_status=missing`，页面显示“不能作为正式准确率”。

数据源：

- `rear_row_metrics.json`
- `sr_ablation_metrics.json`
- `sr_ablation_paper_summary.json`
- `04_论文测评指标结果记录.md`

### 5. 失败案例页

展示内容：

- 漏检：GT 有人但预测没有。
- 误检：预测有人但 GT 没有。
- IDSW：同一 GT 学生切换了模型 track_id。
- 行为错分：行为 GT 与预测不一致。
- 后排 SR 负例：画面变清楚但指标没提升。

数据源：

- `rear_row_failure_cases/`
- `rear_row_metrics.json` 中的错误统计
- 后续建议新增 `failure_cases.json`

## 建议新增前端数据加工脚本

建议新增：`scripts/20_build_frontend_data_bundle.py`

输出目录：

```text
output/frontend_bundle/<case_id>/
  frontend_data_manifest.json
  timeline_students.json
  tracks_sampled.json
  behavior_segments.json
  metrics_summary.json
  ablation_summary.json
  failure_cases.json
  assets/
    pose_behavior_fusion_yolo11x.mp4
    pose_behavior_fusion_yolo11x_preview.jpg
    sr_ablation_contact_sheet.jpg
```

加工规则：

- CSV 转 JSON，减少前端解析差异。
- JSONL 转按 frame/track_id 索引的 JSON。
- 大文件抽样或分片，避免浏览器一次加载几百 MB。
- 所有路径使用相对 bundle 路径。
- 写入 schema 版本号。

## D3 图表数据 schema 建议

### Timeline JSON

```json
{
  "case_id": "front_002_A8",
  "students": [
    {"student_id": "S07", "track_id": 7}
  ],
  "segments": [
    {"student_id": "S07", "track_id": 7, "start_time": 10.0, "end_time": 15.2, "behavior_code": "dk", "confidence": 0.91}
  ]
}
```

### Metrics Summary JSON

```json
{
  "case_id": "front_002_A8",
  "gt_status": "ok",
  "groups": {
    "detection": {"rear_person_recall": 0.82, "AP50": 0.86},
    "pose": {"PCK_0_10": 0.74, "OKS_AP": 0.58},
    "tracking": {"IDF1": 0.87, "IDSW": 2, "HOTA": 0.69},
    "behavior": {"behavior_macro_f1": 0.73, "temporal_mAP_0_5": 0.61},
    "runtime": {"stage_runtime_sec": 343.6, "sr_cache_bytes": 238277298}
  }
}
```

### Failure Cases JSON

```json
{
  "case_id": "front_002_A8",
  "items": [
    {"type": "miss", "frame": 250, "track_id": 7, "bbox": [420,105,455,178], "image": "assets/fail_miss_000250.jpg", "note": "rear row occlusion"}
  ]
}
```

## 前端验收标准

| 验收项 | 标准 |
| --- | --- |
| 视频播放 | 能播放融合视频，能跳转到时间线段起点 |
| 时间线 | 能按学生 ID 展示行为段，颜色对应 8 类行为 |
| 指标页 | formal/proxy/smoke 清楚区分，GT missing 有明显提示 |
| 消融页 | A0/A1/A2/A3/A8 可对比，指标与 contact sheet 同屏 |
| 失败页 | 至少展示漏检、误检、IDSW、行为错分 4 类 |
| 交互 | 点击学生、时间段、指标柱能联动过滤 |
| 文档 | 有数据 schema、运行命令、页面说明、指标解释 |
| 性能 | 单页初次加载不超过可接受时间；大 JSONL 不直接全量加载 |

## UI 建议

- 不要把所有框和标签同时画满屏。默认只显示选中学生或 top-k 重要事件。
- 行为颜色固定：8 类行为全站一致。
- `unlinked` 行为框用灰色虚线，不进入正式时间线。
- 指标页必须显示 `gt_status`，避免把 proxy 当 formal。
- 视频、时间线、座位图三者联动，是前端的核心价值。
