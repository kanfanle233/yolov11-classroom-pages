## 当前最合理的优化方向

### 1. 先补数据，不先堆算法

当前最缺的是：

- 更多有语音的长视频 case
- 更多教师指令事件
- 更大规模的 LLM 语义样本

不是最缺的：

- 继续引入更复杂的对齐公式
- 先做大规模模型替换

### 2. 维持“上下文增强”定位

当前推荐逻辑：

- 有音频时：为视觉行为提供 instruction context。
- 无音频时：正常降级为 visual-only。
- 论文与索引都按这个口径写，避免过度宣称。

### 3. 让现有接口承接更清晰的语音字段

当前可继续加强的对象：

- `verified_events.json`
- `metrics_summary.json`
- `/api/v1/visualization/case_data`
- `index_v2.html`

优化目标：

- 让前端能区分 visual-only 与 audio-assisted。
- 让 bundle 中能明确显示 instruction event 来源。

## 当前最重要的四个动作

1. 为有语音 case 补更稳定的 instruction event 统计。
2. 把 visual-only 与 audio-assisted 在 bundle/BFF 中显式区分。
3. 扩大 LLM 语义样本，不急于把当前 37 对结果写成 benchmark。
4. 在论文和索引中继续把 OCR / 敏感文本保留为未来工作。

## 当前不建议的动作

- 把 OCR 方向混入当前主线。
- 用很少的音频 case 强行下“大规模双模态有效”的结论。
- 在索引里把 D3/BFF 的语音展示方案直接写成已完成。
