> 参考论文：Stanford EDS 2025、Li Yuheng 2026、EDM 2025、BMRN 2025

## 一、当前问题诊断

### 问题矩阵

| 编号 | 问题 | 严重度 | 根因 | 修复方向 |
|---|---|---|---|---|
| P1 | ASR 输出全被质量门控拒绝 | 🔴 | `avg_logprob` 阈值 -0.6 过严 | 打开门控，区分"噪声拒绝"和"低置信保留" |
| P2 | text_score 自匹配（visual fallback 时=1.0）| 🔴 | 兜底事件抄了 semantic_id 作为 query_text | 标注 source，fallback 时 text_score=NA |
| P3 | 语音事件抽取太粗糙 | 🟠 | 仅关键词硬匹配 | 增加语义相似度匹配 |
| P4 | Whisper 输出繁体中文 | 🟡 | Whisper 训练数据繁体占比高 | 简繁转换或 progressive prompt |
| P5 | "双重验证"叙事不实 | 🔴 | ASR 路径从未有效运行 | 重新定位语音角色 |
| P6 | 无声视频 ASR 空跑 | 🟡 | 无预检机制 | 增加音频 VAD/能量预检 |

### 参考论文的解决方式

| 论文 | 语音角色 | 融合方式 | 无声/低质处理 |
|---|---|---|---|
| **Stanford EDS** (2025) | 学生参与度独立指标 | 晚期加权融合 (late fusion) | 无声段用零值填充 |
| **Li Yuheng** (2026) | 教学阶段分类 | LLM 驱动级联管线 | 无语音时跳过语音分支 |
| **EDM 2025** (Hou) | 话语质量辅助特征 | 文本中心注意力融合 | 文本为主要模态 |
| **BMRN** (2025) | 情感特征独立源 | 共享-私有特征解耦 | 模态缺失时用重建损失兜底 |

**共同结论**：所有论文都把语音当作**独立信息维度**，不是"验证者"。语音失效时系统降级为纯视觉/纯文本，不需要假装有多模态。

## 二、语音角色重新定义

### 旧定义（不可行）

```
语音 → 验证视觉是否正确 → "双重验证"
问题：无声视频中语音贡献为零，但叙事声称它在起作用
```

### 新定义（推荐）

```
语音 → 提供视觉无法获取的上下文信息 → "音视频上下文感知"
角色：教师指令 → 行为先验调整
      教学阶段 → 行为分布约束
      关键词触发 → 特定行为置信度增强
无声时：纯视觉模式，不声称有语音贡献
```

### 论文叙事变化

| 维度 | 旧叙事 | 新叙事 |
|---|---|---|
| 标题关键词 | "双重验证框架" | "音视频上下文感知" |
| ASR 贡献 | "跨模态一致性验证" | "教学场景上下文增强" |
| 无声处理 | "质量门控 + 视觉兜底" | "模态缺失鲁棒降级" |
| 创新点 | "UQ 驱动自适应对齐 + 双重验证" | "课堂特化的音视频互补 + 模态容错" |

## 三、分阶段实施计划

### Phase 1：诊断与修复基础管道（1-2 小时）

**目标**：让 ASR 在有语音的视频上能产出有效事件

#### 1.1 修复质量门控（P1）

修改 `scripts/06_asr_whisper_to_jsonl.py`：

```python
# 当前逻辑（过严）
if avg_logprob < -0.6:  # 全部拒绝
    reject()

# 改为分层门控
if avg_logprob < -2.0 or no_speech_prob > 0.8:
    reject("high_noise")       # 确定噪声，拒绝
elif avg_logprob < -0.6:
    accept_with_flag("low_conf")  # 低置信但保留，标记 source="asr_low_conf"
else:
    accept("normal")           # 正常接受
```

**参考**：Stanford EDS 使用 Wav2Vec2.0 的置信度分数，不做硬拒绝，而是在融合时用低权重衰减。

#### 1.2 增加音频预检（P6）

在 ASR 之前增加快速预检：

```python
def check_audio_quality(wav_path):
    """返回音频质量等级"""
    audio = read_wav(wav_path)
    rms = compute_rms(audio)
    speech_prob = vad_detect(audio)  # 使用简单的能量VAD
    
    if rms < 50:       return "silence"      # 静音，跳过
    if rms < 150:      return "quiet"         # 安静，可以尝试
    if speech_prob < 0.3: return "no_speech"   # 非语音
    return "normal"                             # 正常
```

**有语音时**：正常走 ASR → 事件抽取 → 上下文增强
**静音时**：跳过 ASR，直接进入纯视觉模式（不再生成假事件）

#### 1.3 修复简繁体（P4）

在 Whisper 输出后增加简繁转换：

```python
# 方案A：使用 opencc 库
import opencc

text_simplified = converter.convert(text)

# 方案B（备选，无额外依赖）：简单字符映射
# 仅覆盖常见的繁简对应
```

### Phase 2：重新设计事件抽取与融合（2-3 小时）

**目标**：从 ASR 文本中抽取有意义的上下文事件，实现音视频互补

#### 2.1 重新设计事件类型（P3）

旧事件类型（太抽象，无法和视觉关联）：
```
"read" → query_text="read" → 自匹配=1.0
```

新事件类型（具体、可操作、独立于视觉）：

```python
TEACHER_INSTRUCTION_PATTERNS = {
    "open_book":    ["翻开(课?本|书)", "打开(课?本|书)", "看(课?本|书)第"],
    "start_write":  ["开始写", "写下来", "做笔记", "记下来", "写在"],
    "look_board":   ["看黑板", "看前面", "看屏幕", "抬头"],
    "raise_hand":   ["谁来回答", "举手", "哪位同学"],
    "read_aloud":   ["一起读", "朗读", "读一遍", "跟着读"],
    "discuss":      ["讨论", "交流", "同桌", "小组"],
    "stop":         ["停笔", "停下来", "放下笔", "不要写了"],
}

# 匹配逻辑：不是硬匹配，而是计算语义相似度
def extract_instruction_event(text, timestamp):
    for event_type, patterns in TEACHER_INSTRUCTION_PATTERNS.items():
        for pattern in patterns:
            if re.search(pattern, text):  # 或用 fuzzy matching
                return {
                    "event_type": event_type,  # e.g., "open_book"
                    "query_text": text,
                    "timestamp": timestamp,
                    "confidence": compute_match_score(pattern, text),
                    "source": "asr_instruction"
                }
    return None  # 无匹配则不出事件
```

**参考**：Stanford EDS 使用关键词分类器（keyword-based classifier），比纯规则匹配更鲁棒。Li Yuheng 用 LLM 做教学阶段分类，效果更好但开销更大。

#### 2.2 重新设计融合逻辑（P2 + P5 核心修复）

```python
def context_aware_scoring(visual_action, asr_events, time_window):
    """
    语音作为上下文增强器，不是验证器。
    
    返回值：调整后的行为置信度
    """
    base_confidence = visual_action.confidence
    
    # 查找时间窗口内的教师指令
    relevant_events = [e for e in asr_events 
                       if abs(e.timestamp - visual_action.timestamp) < time_window]
    
    if not relevant_events:
        return base_confidence, {"source": "visual_only"}
    
    # 上下文增强：教师指令提升相关行为的置信度
    boost = 0.0
    for event in relevant_events:
        if event.type == "open_book" and visual_action.behavior in ["dk", "read"]:
            boost += 0.10  # 教师说"翻开课本"→读书行为置信度+10%
        elif event.type == "start_write" and visual_action.behavior in ["dx", "write"]:
            boost += 0.10
        elif event.type == "look_board" and visual_action.behavior in ["tt", "listen"]:
            boost += 0.10
        elif event.type == "stop" and visual_action.behavior in ["dx", "write"]:
            boost -= 0.15  # 教师说"停笔"→写字行为置信度-15%
    
    adjusted_conf = min(1.0, max(0.0, base_confidence + boost))
    
    return adjusted_conf, {
        "source": "audio_visual_context",
        "base_confidence": base_confidence,
        "adjustment": boost,
        "trigger_events": [e.type for e in relevant_events]
    }
```

**关键差异**：旧逻辑是"语音验证视觉对错"（text_score vs visual_score），新逻辑是"语音调整行为先验"（boost/dampen confidence）。这不声称"验证"，只声称"增强"。

#### 2.3 修改 verifier 输出 schema

```json
// 旧 schema（有问题的）
{
    "label": "match",
    "evidence": {
        "visual_score": 0.93,
        "text_score": 1.0,      // ← 自匹配假值
        "uq_score": 0.11
    }
}

// 新 schema（修复后）
{
    "label": "high_confidence",  // 不再用 match/mismatch
    "confidence": 0.89,
    "context_source": "audio_visual",  // 或 "visual_only"
    "evidence": {
        "visual_score": 0.93,
        "audio_context_boost": 0.10,   // ← 来自教师指令
        "audio_events": ["open_book"],  // 触发的事件类型
        "uq_score": 0.11
    }
}
```

修改 `check_pipeline_contract_v2.py`：
- 不再断言 `verified_events` 的 `text_score` 字段
- 新增检查 `context_source` 字段
- 允许 `visual_only`（无声时的正常状态）

## 六、参考实现

### Stanford EDS 晚期融合策略（最简可行方案）

```
audio_score = keyword_classifier(audio_transcript)  # 0-1
visual_score = behavior_detector(video_frame)       # 0-1
fused_score = alpha * visual_score + (1-alpha) * audio_score
if audio_available == False:
    fused_score = visual_score  # 降级
```

### Li Yuheng LLM 驱动方案（长期目标）

```
transcript → LLM → teaching_phase ∈ {lecture, discussion, practice, Q&A}
teaching_phase → behavior_prior_adjustment
```

我们当前选择 Stanford EDS 路线（简单、可快速验证），LLM 方案列为 future work。
